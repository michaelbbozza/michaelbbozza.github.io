Class is now in session!--or at least in Ruby. Classes can be used to group together multiple methods, or "bundle" them together. I will show examples on this later on in this blog to help illustrate what I mean. Classes are used primarily to organize the data and make programming large amounts of code a lot simpler than constantly going through method by method and changing everything, sometimes repeating the same code. (Note: #1 Rule: D.R.Y. Don't Repeat Yourself) Methods with no Class implemented generally rely on themselves, rather than the whole block. Methods are great, until you want the method to work on multiple objects, causing you to write a million "if/else" or "case" or other loop statements. This is where using a Class can really be helpful. It can reduce the amount of loops and conditionals in your code, as well as work for multiple items, without retyping the same code.

class Name
end

After creating your Class, you can define your methods inside the Class structure. "Initialize" should be the first method you create. This method should be defined to take an argument (or number of arguments) that can later be called (or initialized) and run your code within the Class. Let's say you create the method to take 3 arguments, but these arguments will have different values for different things. This is when you will create "Instance Variables". Instance variables are used when you want to create a different results for a different object. For you gamers out there, let's say one of your arguments is "health" but you have 10 characters with different health limits (1 is 150 HP, another is 100 HP, and so on). This is when you would create an Instance Variable using @ (at symbol) and make it equal to "health".
Example: @health = health
This will allow you to assign different values of "health" to different characters.

class Name
def initialize(name)
@name = name
@name = gets.chomp
puts "Hello" + @name + "."
end
end

This example shows the class "Name" with the method "initialize" taking the argument "name". "@name" is the instance variable being used for anything that is associated with "name". "@name = gets.chomp" refers to acquiring the user's name that they input themselves. The fourth line tells the program to print "Hello" and then the user's name. The first "end" closes the method "initialize" and the second "end" closes the class "Name". Simply put, when ran, this program will ask you for your name, and respond with "Hello (your name)."

In short:

  • Classes are used to minimize the amount of retyping and repeating code in a program.
  • Classes are extremely useful in the fact that they can be used for large amounts of data or multiple instances of data.
  • Use "Instance Variables" to use the variable for different methods to produce a different result. (Such as the "health" example in the last paragraph)
  • "Instance Variables" can be used at anytime throughout the class.
